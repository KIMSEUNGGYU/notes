# 기본 개념

개발자를 위한 효과적인 글쓰기의 기본 원칙을 소개합니다.

## 왜 글쓰기가 중요한가?

개발자에게 글쓰기는 코딩만큼 중요한 능력입니다. 좋은 코드를 작성하는 것만큼이나, 그 코드를 다른 사람에게 설명하고 문서화하는 능력이 중요합니다.

### 글쓰기가 필요한 순간들

- **코드 리뷰**: 변경 사항을 명확하게 설명
- **Pull Request**: 왜 이 변경이 필요한지 설득
- **기술 문서**: 시스템 아키텍처나 API 사용법 설명
- **이슈 보고**: 버그나 개선사항을 구체적으로 전달
- **코드 주석**: 복잡한 로직의 의도 설명
