# 실전 사례

실제 개발 현장에서 활용할 수 있는 글쓰기 예제를 소개합니다.

## Pull Request 작성하기

효과적인 PR 설명은 코드 리뷰 시간을 단축시키고, 변경사항에 대한 이해도를 높입니다.

### 좋은 PR의 구조

```markdown
## 변경 사항
- 사용자 인증 로직 개선
- Redis 캐싱 추가

## 왜 필요한가?
기존 인증 방식은 매 요청마다 DB를 조회하여 성능 이슈가 있었습니다.

## 해결 방법
Redis를 활용한 세션 캐싱으로 DB 조회 횟수를 90% 감소시켰습니다.

## 테스트 방법
1. 로그인 후 `/api/user` 호출
2. Redis CLI로 캐시 확인: `keys session:*`
```
